# Сыпачев Иван КМБ-2 Лабараторная №7
# Задание 1
## Задача 10
### Текст задачи
Задача 1 - В файле найти квадрат разности максимального и минимального элементов.
Задача 2 - Вычислить сумму нечётных элементов файла.
Задача 3 - Создать новый текстовый файл, каждая строка которого содержит последний символ
соответствующей строки исходного файла.
Задача 4 - Найти количество удвоенных нечетных чисел среди компонент файла.
Задача 5 - Файл содержит сведения об игрушках: название игрушки, ее стоимость в рублях и возрастные
границы (например, игрушка может предназначаться для детей от двух до пяти лет). Получить
название самой дешевой игрушки.
### Алгоритм решения
Методы GenerateTask1File и GenerateTask2File: Используют объект Random для создания текстовых файлов. Первый генерирует столбец случайных чисел, второй — матричную структуру (строки и столбцы), заполненную целыми числами.
Метод GenerateBinaryRandomFile: Реализует запись данных в низкоуровневом формате с помощью BinaryWriter, сохраняя последовательность целых чисел в компактном двоичном виде.
Метод DiffSquare: Выполняет поиск максимального и минимального значений в текстовом файле. Итоговым результатом является квадрат их разности. Встроена проверка на пустоту файла и корректность парсинга строк.
Метод SumOddElements: Считывает данные, разделенные пробелами, и вычисляет сумму всех нечетных чисел в файле.
Метод ExtractLastCharacters: Обрабатывает текстовый файл построчно, извлекая последний символ каждой строки и сохраняя результат в новый выходной файл, формируя таким образом строку «окончаний».
Методы PrintBinaryNumbers и CountDoubleOddsBinary: Используют BinaryReader для последовательного чтения данных. Программа перемещается по потоку (BaseStream), пока не достигнет конца файла. В процессе выполняется либо вывод чисел на экран, либо подсчет «дважды нечетных» чисел (четных, половина которых нечетна).
Вспомогательный метод PrintTaskFile: Обеспечивает быструю визуализацию содержимого текстовых файлов в консоли.
Метод ValidateFile: Централизованный механизм проверки существования файла по указанному пути. В случае отсутствия объекта выводит информативное сообщение и предотвращает выполнение операций, которые могли бы привести к критической ошибке.

# Задание 2
## Задача 10
### Текст задачи
Задача 1 - Составить программу, которая в списке L из каждой группы подряд идущих одинаковых
элементов оставляет только один
Задача 2 - Построить двунаправленный список L по однонаправленному списку L1
Задача 3 - Есть перечень факультативов в учебном заведении. Студенты группы записались на некоторые
из этих факультативов. Известно для каждого студента, на какие факультативы он записан.
Определить:
• на какие факультативы из перечня ходят все студенты группы;
• на какие факультативы из перечня ходит хотя бы один студент группы;
• на какие факультативы из перечня не ходит ни один из студентов группы?
Задача 4 - Файл содержит текст на русском языке. Напечатать в алфавитном порядке символы, которые
встречаются хотя бы однажды в словах с нечётными номерами (нумерацию вести с 1). 
Задача 5 - На автозаправочных станциях (АЗС) продается бензин с маркировкой 92, 95 и 98. В городе N был
проведен мониторинг цены бензина на различных АЗС. Напишите программу, которая будет
определять для каждого вида бензина, сколько АЗС продают его дешевле всего.
На вход программе в первой строке подается число данных N о стоимости бензина. В каждой из
последующих N строк находится информация в следующем формате:
<Компания><Улица><Марка><Цена>
где <Компания> – строка, состоящая не более, чем из 20 символов без пробелов, <Улица> –
строка, состоящая не более, чем из 20 символов без пробелов, <Марка> – одно из чисел – 92, 95
или 98, <Цена> – целое число в диапазоне от 1000 до 3000, обозначающее стоимость одного
литра бензина в копейках.
<Компания> и <Улица>, <Улица> и <Марка>, а также <Марка> и <Цена> разделены ровно одним
пробелом. Пример входной строки:
Синойл Цветочная 95 2250
Программа должна выводить через пробел 3 числа – количество АЗС, продающих дешевле всего
92-й, 95-й и 98-й бензин соответственно. Если бензин какой-то марки нигде не продавался, то
следует вывести 0.
### Алгоритм решения
Метод RemoveDuplicates: Осуществляет поиск и удаление дубликатов в списке чисел с плавающей точкой. Алгоритм сравнивает соседние элементы и производит удаление при обходе списка с конца, что предотвращает пропуски элементов при изменении размера коллекции.
Метод BuildDouble: Трансформирует одномерный список в двусвязный (LinkedList<double>), сохраняя порядок следования элементов.
Метод GenerateElectivesData: Автоматически формирует случайный набор доступных учебных дисциплин и имитирует выбор нескольких студентов, используя Random и HashSet для исключения повторов внутри одного выбора.
Метод Electives: Проводит глубокий анализ посещаемости с помощью теоретико-множественных операций:
IntersectWith: Вычисляет пересечение множеств для поиска общих для всех студентов курсов.
UnionWith: Формирует полное множество выбранных курсов.
ExceptWith: Выявляет невостребованные дисциплины путем вычитания выбранных курсов из общего списка.
Метод ProcessOddWordsChars: Выполняет разбор текстового файла, извлекает каждое нечетное слово и формирует отсортированный список всех уникальных букв, встречающихся в этих словах.
Метод GenerateGasDataFile: Генерирует файл со структурированной информацией (название компании, улица, марка топлива, цена) для имитации базы данных автозаправочных станций.
Метод SolveGasTask: Проводит статистический анализ файла. С помощью словаря списков (Dictionary<int, List<int>>) группирует цены по маркам бензина (92, 95, 98), находит минимальную стоимость для каждой категории и подсчитывает количество станций с такой ценой.
Метод ValidateFile: Централизованный механизм проверки существования файла по указанному пути. В случае отсутствия объекта выводит информативное сообщение и предотвращает выполнение операций, которые могли бы привести к критической ошибке.
