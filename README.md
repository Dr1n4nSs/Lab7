# Сыпачев Иван КМБ-2 Лабараторная №7
# Задание 1
## Задача 10
### Текст задачи
Задача 1 - В файле найти квадрат разности максимального и минимального элементов.
Задача 2 - Вычислить сумму нечётных элементов файла.
Задача 3 - Создать новый текстовый файл, каждая строка которого содержит последний символ
соответствующей строки исходного файла.
Задача 4 - Найти количество удвоенных нечетных чисел среди компонент файла.
Задача 5 - Файл содержит сведения об игрушках: название игрушки, ее стоимость в рублях и возрастные
границы (например, игрушка может предназначаться для детей от двух до пяти лет). Получить
название самой дешевой игрушки.
### Алгоритм решения
Метод CalculateMaxMinDiffSquare: Считывает все строки файла и находит экстремумы (максимум и минимум) среди корректно распознанных вещественных чисел. В качестве результата возвращает квадрат разности между найденными значениями.
Метод SumOddElements: Выполняет фильтрацию целых чисел в файле, суммируя только те, что являются нечетными.
Метод CountDoubleOdds: Реализует специфическую логическую проверку: подсчитывает количество четных чисел, которые при делении на два дают нечетный результат.
Метод ExtractLastCharacters: Считывает содержимое одного файла и формирует новый, записывая в каждую строку только последний символ из соответствующей строки исходного текста. Пустые строки обрабатываются без ошибок.
Структура Toy: Описывает объект «Игрушка» с полями для названия, цены и возрастных ограничений. Инкапсуляция реализована через публичные свойства.
Метод CreateInitialFile: Демонстрирует использование XmlSerializer для сохранения массива структур в формате XML. Создает набор тестовых данных и записывает их в поток файла.
Метод GetCheapestToyName: Выполняет десериализацию данных из XML-файла обратно в массив объектов и находит название игрушки с минимальной стоимостью путем линейного поиска.
Внутренний статический метод ValidateFile обеспечивает проверку существования целевых файлов перед началом работы. Если файл отсутствует, программа выбрасывает исключение FileNotFoundException, предотвращая аварийное завершение работы методов обработки.

# Задание 2
## Задача 10
### Текст задачи
Задача 1 - Составить программу, которая в списке L из каждой группы подряд идущих одинаковых
элементов оставляет только один
Задача 2 - Построить двунаправленный список L по однонаправленному списку L1
Задача 3 - Есть перечень факультативов в учебном заведении. Студенты группы записались на некоторые
из этих факультативов. Известно для каждого студента, на какие факультативы он записан.
Определить:
• на какие факультативы из перечня ходят все студенты группы;
• на какие факультативы из перечня ходит хотя бы один студент группы;
• на какие факультативы из перечня не ходит ни один из студентов группы?
Задача 4 - Файл содержит текст на русском языке. Напечатать в алфавитном порядке символы, которые
встречаются хотя бы однажды в словах с нечётными номерами (нумерацию вести с 1). 
Задача 5 - На автозаправочных станциях (АЗС) продается бензин с маркировкой 92, 95 и 98. В городе N был
проведен мониторинг цены бензина на различных АЗС. Напишите программу, которая будет
определять для каждого вида бензина, сколько АЗС продают его дешевле всего.
На вход программе в первой строке подается число данных N о стоимости бензина. В каждой из
последующих N строк находится информация в следующем формате:
<Компания><Улица><Марка><Цена>
где <Компания> – строка, состоящая не более, чем из 20 символов без пробелов, <Улица> –
строка, состоящая не более, чем из 20 символов без пробелов, <Марка> – одно из чисел – 92, 95
или 98, <Цена> – целое число в диапазоне от 1000 до 3000, обозначающее стоимость одного
литра бензина в копейках.
<Компания> и <Улица>, <Улица> и <Марка>, а также <Марка> и <Цена> разделены ровно одним
пробелом. Пример входной строки:
Синойл Цветочная 95 2250
Программа должна выводить через пробел 3 числа – количество АЗС, продающих дешевле всего
92-й, 95-й и 98-й бензин соответственно. Если бензин какой-то марки нигде не продавался, то
следует вывести 0.
### Алгоритм решения
Метод RemoveDuplicates: Удаляет дубликаты из предварительно отсортированного списка целых чисел. Алгоритм осуществляет проход с конца коллекции к началу, сравнивая текущий элемент с предыдущим, что позволяет безопасно удалять элементы по индексу без нарушения логики цикла.
Метод BuildDouble: Преобразует стандартный динамический список List<int> в двусвязный список LinkedList<int>, последовательно добавляя каждый элемент в конец новой структуры.
Метод Electives: Использует класс HashSet<string> для анализа посещаемости курсов студентами. Реализует три ключевые операции:
Пересечение (IntersectWith): Для поиска курсов, на которые ходят абсолютно все студенты.
Объединение (UnionWith): Для формирования списка всех задействованных курсов (хотя бы один студент).
Разность (ExceptWith): Для выявления курсов из общего списка, которые не выбрал ни один студент.
Метод ProcessOddWordsChars: Парсит текстовый файл, разделяя его на слова по пробельным символам. Выбирает слова на нечетных позициях, извлекает из них уникальные буквы (в нижнем регистре) и выводит их в консоль в алфавитном порядке.
Метод SolveGasTask: Анализирует данные о ценах на топливо. Использует Dictionary<int, List<int>> для группировки цен по маркам бензина (92, 95, 98).
Алгоритм поиска минимума: Для каждой марки топлива программа находит минимальную цену, а затем подсчитывает количество заправок, предлагающих эту цену. Результат выводится в виде трех итоговых чисел.
Содержит инкапсулированные методы для безопасного чтения целых чисел из консоли с валидацией (ReadInt) и форматированного вывода коллекций (PrintIntList, PrintSet).
